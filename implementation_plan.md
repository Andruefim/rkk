# Реализация Гомеостатического Каузального Контроля (Active Inference)

Текущая архитектура пытается использовать максимизацию информационного выигрыша (Curiosity/EIG) для решения задачи моторного контроля, что приводит к нестабильности (агент пытается максимизировать ошибку предсказания, совершая хаотичные действия). Новый план переносит фокус на поддержание гомеостаза (выживания) через Принцип Свободной Энергии (Active Inference).

## User Review Required

> [!WARNING]
> Это фундаментальное изменение архитектуры принятия решений агента. Существующий `GoalImagination`, который стимулировал агента "удивлять себя", будет переведен на уровень Системы 2 (навигация/планирование), а базовый моторный контроль (Система 1) будет жестко привязан к поддержанию баланса.

## Open Questions

> [!IMPORTANT]
> 1. **Применение внешних сил (Perturbations):** Для обучения рефлексов в фазе `fixed_root` нам нужно толкать агента. Поддерживает ли ваш кастомный `env` в PyBullet применение внешних импульсов (`applyExternalForce`), или нам лучше симулировать толчок через резкое изменение `com_x/com_y` на стороне физического движка?
> 2. **Доступ к `intent_*` градиентам:** В `CausalGNNCore` действия подаются на вход. Чтобы сделать backprop в действия, нам потребуется прокинуть градиенты от предсказания `X_{t+k}` до вектора `a_t`. Нет ли архитектурных ограничений в вашем пайплайне `LeWM`, которые мешают брать градиенты по входу GNN?

## Proposed Changes

---

### Phase 1: Core Active Inference Engine (System 1)

Мы создадим новый модуль, который будет вычислять моторные команды не через случайный семплинг или CEM, а через инверсию градиентов (Backpropagation to Actions).

#### [NEW] `backend/engine/active_inference.py`
- Создание класса `HomeostaticController`.
- **Вход:** Текущее состояние $X_t$, Causal GNN (World Model), Homeostatic Prior (целевые значения, например `posture=1.0`, `com_z=0.45`).
- **Логика:**
  1. Инициализирует вектор действий $a_t$ (из предыдущего шага или нулями).
  2. Прогоняет $a_t$ через GNN (`forward_dynamics_seq` на H шагов вперед).
  3. Вычисляет функцию потерь $L = MSE(X_{pred\_target}, X_{prior\_target})$.
  4. Выполняет $N$ шагов градиентного спуска по вектору $a_t$ ($\nabla_a L$).
  5. Возвращает оптимизированный вектор `intent_*` как моторную команду.

#### [MODIFY] `backend/engine/causal_gnn.py`
- Убедиться, что `forward_dynamics` и `forward_dynamics_seq` могут пропускать градиенты до входного тензора `a` (сейчас `a` может быть отвязан от графа вычислений, если не `requires_grad=True`).

---

### Phase 2: Reflex Learning in Fixed Root

Изменение фазы `fixed_root` так, чтобы она не была просто пассивным закреплением с рандомным бабблингом, а стала фазой "обучения реакциям на возмущения".

#### [MODIFY] `backend/engine/features/simulation/mixin_skills.py` (или `simulation_main.py`)
- Добавить логику: если `self._fixed_root_active == True`:
  - Каждые $K$ тиков (например, 50) генерировать случайное возмущение (например, задавать целевое отклонение торса или прикладывать виртуальную силу).
  - Использовать `HomeostaticController` для генерации компенсирующих `intent_*`.
  - GNN будет обучаться связывать "ошибка `posture` -> компенсация `intent_hip` -> возврат `posture` в норму".

---

### Phase 3: System 2 vs System 1 Separation

Рефакторинг Интринсивной цели так, чтобы любопытство не мешало стоять на ногах.

#### [MODIFY] `backend/engine/intristic_objective.py`
- Переписать `GoalImagination`:
  - Сейчас он выбирает цель максимизирующую $abs(X_{t+H} - X_t)$.
  - Изменить логику: Система 2 (Curiosity) активируется *только если* Система 1 (Homeostasis) стабильна (например, средний `posture` > 0.8 за последние 100 тиков).
  - Если стабильна, `GoalImagination` генерирует цель не в пространстве всех переменных, а только в пространстве навигации/позиции (`com_x`, `com_y`, `gaze_yaw`), передавая эту цель как новый Приор для `HomeostaticController`.
- Изменить общий расчет награды: `Reward = Homeostasis_Factor * EIG`. Упавший агент не должен получать награду за "удивление".

---

### Phase 4: Trajectory-Level Causal Learning

Обучение GNN на паттернах стабильности.

#### [MODIFY] `backend/engine/causal_graph.py`
- При обучении GNN на последовательностях из `EpisodicMemory`, добавить взвешивание лосса (Reward-Weighted Regression).
- Эпизоды, где агент простоял долго (высокий `posture`), получают больший вес или используются как anchor в contrastive loss, чтобы латентное пространство (JEPA) кластеризовало стабильные и нестабильные состояния далеко друг от друга.

2. **Transition to Free**: Дождаться окончания 2000 тиков. Агент должен быть способен использовать выученные рефлексы из `HomeostaticController` для удержания баланса в свободном падении, значительно превышая текущие результаты (где он падает за 1.5 тика).
